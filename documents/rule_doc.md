# Quy tắc đọc: Chỉ cần quan tâm những thứ liên quan đến Code, Dữ liệu, ko cần quan tâm đến Báo cáo tiểu luận

# Căn cứ theo CLO của môn học

## CLO1

Áp dụng được các lý thuyết, phương pháp và quy trình cơ bản trong khoa học dữ liệu để xử lý, phân tích và trực quan hóa dữ liệu; bao gồm các bước thu thập, làm sạch, biến đổi dữ liệu, cũng như trích xuất đặc trưng và xây dựng mô hình bằng các kỹ thuật học máy.

---

# Project đánh giá kết thúc học phần

Project đánh giá kết thúc học phần sẽ gồm các thành phần:

- Xác định bài toán thực tế;
- Thu thập hoặc chọn dữ liệu phù hợp;
- Làm sạch và chuẩn bị dữ liệu;
- Phân tích thống kê mô tả và phát hiện ngoại lai;
- Trực quan hóa dữ liệu bằng Python;
- Xây dựng ít nhất một mô hình hoặc thuật toán phù hợp;
- Đánh giá kết quả bằng độ đo thích hợp;
- Trình bày và thảo luận kết quả.

---

# Sản phẩm bắt buộc

- **Báo cáo tiểu luận:** 12–20 trang A4, trình bày theo cấu trúc khoa học.
- **Mã nguồn Python:** notebook `.ipynb` hoặc file `.py`, chạy được.
- **Dữ liệu sử dụng:** file `.csv`, `.xlsx`, `.json` hoặc link nguồn dữ liệu.
- **Slide báo cáo:** 8–12 trang.
- **Video ngắn hoặc demo trực tiếp:** 5–10 phút.
- **Phụ lục:** mô tả cách chạy chương trình, thư viện cần cài đặt.

---

# Cấu trúc báo cáo thống nhất

1. Giới thiệu bài toán thực tế
2. Mục tiêu và câu hỏi phân tích
3. Mô tả dữ liệu
4. Thu thập dữ liệu / nguồn dữ liệu
5. Làm sạch và tiền xử lý dữ liệu
6. Phân tích thống kê mô tả
7. Phát hiện ngoại lai
8. Trực quan hóa dữ liệu
9. Xây dựng mô hình / thuật toán
10. Đánh giá kết quả
11. Kết luận và hướng phát triển

---

# Yêu cầu kỹ thuật tối thiểu

Mỗi dự án phải có đủ:

1. Ít nhất 01 bảng dữ liệu thực tế;
2. Ít nhất 03 biểu đồ trực quan;
3. Có xử lý dữ liệu thiếu hoặc dữ liệu không nhất quán;
4. Có kiểm tra/phân tích ngoại lai;
5. Có ít nhất 01 bài toán mô hình hóa, dự đoán hoặc khuyến nghị;
6. Có đánh giá bằng độ đo phù hợp:
   - **Bài toán phân loại:** Accuracy, Precision, Recall, F1-score;
   - **Bài toán hồi quy:** MSE, MAE;
   - **Bài toán khuyến nghị:** có thể dùng độ tương đồng, sai số dự đoán, so sánh dự đoán với giá trị thực nếu có;
7. Có giải thích kết quả, không chỉ dừng ở việc chạy code.
